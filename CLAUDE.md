# 資金通帳 PWA — 開発メモ

## デプロイ時のチェックリスト

### Service Worker キャッシュバージョンの更新

**デプロイのたびに必ず以下の 1 行を更新すること。**

`sw.js` の先頭:
```js
const CACHE = "cashflow-v7";   // ← ここのバージョンを上げる
```

#### バージョンの上げ方

| 変更内容 | 例 |
|---|---|
| HTML / JS / CSS を変更したとき | `cashflow-v7` → `cashflow-v8` |
| アイコン・マニフェストのみ変更 | 同上 |
| Service Worker 自体の変更 | 同上 |

> バージョンを上げないと、既存ユーザーのブラウザが古い SW のまま動き続け、
> 更新バナーが表示されない。

#### 仕組み

- 新 SW がインストールされると `install` → `installed/waiting` 状態で停止する
- ページ側が `updatefound` イベントを検知 → 「新しいバージョンがあります [更新]」バナーを表示
- ユーザーが「更新」をタップ → `SKIP_WAITING` メッセージを送信 → SW が即時 activate → ページリロード
- activate 時に旧バージョン (`cashflow-v*` で名前が異なるもの) のキャッシュを自動削除

#### 戦略

| リソース | 戦略 |
|---|---|
| `index.html` | **Network-first** (常に最新を取得、オフライン時のみキャッシュ) |
| アイコン・manifest | **Cache-first** (高速配信、SW 更新時に入れ替え) |
